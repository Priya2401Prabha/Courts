Description

Based on historical sales and manufacturing trends, this Stock Forecast Report helps you calculate the average daily usage, forecasted demand, and shows how many days your current stock can cover. It also suggests the required quantity while keeping buffer stock in mind. If you use reorder rules, the system directly gives you the required quantity from your warehouse settings. From the same screen, you can create a Request for Quotation or generate a Manufacturing Order, making stock planning quick and easy.

Key Highlights

  • Forecast Methods:Select from history-based consumption or reorder rules for accurate planning.
  • Product Selection: Generate forecasts for specific products or entire product categories.
  • Average Daily Consumption:Automatically computes usage from sales orders and manufacturing consumption.
  • Forecasted Demand: Products required stock over a selected horizon.
  • Available Stock Check: Shows real-time available stock (virtual availability).
  • Required Quantity Calculation: Determines stock shortfall considering buffer stock.
  • Service Days Calculation:Shows how many days stock can serve current demand.
  • Reorder Rule Integration:Considers min/max quantities from warehouse orderpoints.
  • One-Click Actions:Create RFQs or manufacturing orders directly from results.
  • Seamless Integration:Works with purchase, manufacturing, and reordering rules.

Screenshots

The report wizard can be accessed through Inventory → Reporting → Forecast, and the data is retrieved based on the selected product or product category.

Forecast Method: Choose the calculation method for forecasting:
– Reorder Rule: Based on product reordering rules (min/max quantity).
– History Based: Based on past sales and/or manufacturing consumption.

Date Range: Define the period from which historical consumption data will be collected. Options include Last 3 Months, Last 6 Months, Last 12 Months, or Custom.

From / To: If Custom is selected in Date Range, you can specify the start date (From) and end date (To) manually.

Horizon Days: Define how many days ahead the system should forecast the demand and procurement needs.

Buffer Stock (%): Set a buffer stock percentage to maintain safety stock above the calculated requirement. The system will adjust the required quantity accordingly.

Type of Consumption: Specify which type of consumption should be considered for history-based calculations:
– Sales: Only confirmed sales orders.
– Manufacturing: Only consumption in manufacturing.
– Both: Combine sales and manufacturing consumption.

The user must select a forecast method. If History Based is chosen, additional data needs to be entered, whereas if Reorder Rule is selected, no additional data is required.

If the user selects History Based, the report provides calculations for average daily consumption, forecasted demand, current net quantity, sold quantity, MRP consumed quantity, serviceable days, and required quantity.

If the user selects Reorder Rule, the report provides calculations for forecasted demand, current net quantity and required quantity.

From the report, the user can directly generate an RFQ or create a Manufacturing Order (MO).

If a product does not have a vendor assigned, a validation error will be raised.
